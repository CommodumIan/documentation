---

Allows creating new markets on the network

```graphql
type NewMarketInput {
  instrument: InstrumentConfigurationInput!
  decimalPlaces: Int!
  riskParameters: RiskParametersInput!
  metadata: [String!]
  priceMonitoringParameters: PriceMonitoringParametersInput
  continuousTrading: ContinuousTradingInput
  discreteTrading: DiscreteTradingInput
  commitment: NewMarketCommitmentInput
}
```

### Fields

#### `instrument` ([`InstrumentConfigurationInput!`](/docs/graphql/inputs/instrument-configuration-input))

New market instrument configuration

#### `decimalPlaces` ([`Int!`](/docs/graphql/scalars/int))

Decimal places used for the new market

#### `riskParameters` ([`RiskParametersInput!`](/docs/graphql/inputs/risk-parameters-input))

New market risk configuration

#### `metadata` ([`[String!]`](/docs/graphql/scalars/string))

Metadata for this instrument, tags

#### `priceMonitoringParameters` ([`PriceMonitoringParametersInput`](/docs/graphql/inputs/price-monitoring-parameters-input))

Price monitoring configuration

#### `continuousTrading` ([`ContinuousTradingInput`](/docs/graphql/inputs/continuous-trading-input))

A mode where Vega try to execute order as soon as they are received. Valid only if discreteTrading is not set

#### `discreteTrading` ([`DiscreteTradingInput`](/docs/graphql/inputs/discrete-trading-input))

Frequent batch auctions trading mode. Valid only if continuousTrading is not set

#### `commitment` ([`NewMarketCommitmentInput`](/docs/graphql/inputs/new-market-commitment-input))

The liquidity commitment submitted with the new market
